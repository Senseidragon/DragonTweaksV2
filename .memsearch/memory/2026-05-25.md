## NeoForge Event Bus Routing — Hard Rule

**Title:** NeoForge event bus routing — mod bus vs game bus  
**Type:** Architecture rule  
**Enforcement level:** Hard — violation silently fails; handler is never called, no exception thrown  
**Intent triggers:** event bus, mod bus, game bus, registerEventBus, FMLCommonSetupEvent, FMLClientSetupEvent, ModConfigEvent, ServerStartingEvent, NeoForge.EVENT_BUS, @SubscribeEvent  
**Rule or fact:** Mod-lifecycle events (FMLCommonSetupEvent, FMLClientSetupEvent, ModConfigEvent) must be registered on the **mod event bus** (passed to `registerEventBus`). Game/world events (ServerStartingEvent, block/item/entity registration events) must be registered on the **game bus** (NeoForge.EVENT_BUS). Crossing the buses silently drops the handler — no compile error, no runtime exception. When adding or moving a subscriber, verify bus assignment first.
